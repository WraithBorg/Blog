## 单一职责原则
+ `Single Responsibility Principle 简称SRP`
+ *定义* 应该有且仅有一个原因引起类的变更
+ 优点
    + 类的复杂性降低
    + 可读性提高
    + 可维护性提高
    + 变更引起的风险降低
+ 适用于 接口，类和方法

## 里氏替换原则
+ *定义* 只要有父类出现的地方，子类就可以出现，而且替换为子类，也不会产生任何异常和错误，使用者可能不知道是父类还是子类
+ *设计*
    + 子类必须完全实现父类的方法
    + 子类可以有自己的方法和属性
    + 覆盖或实现父类方法时，输入参数可以被放大
    + 覆写或实现父类的方法时输出结果可以被缩小
