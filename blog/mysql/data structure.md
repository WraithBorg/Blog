### B+树
```
特点
    所有记录节点按键值大小顺序放到同一层的叶子节点上，由各个叶子节点指针进行连接
    高扇出性，因此数据库中B+树一般高度在2到4层，意味着查找某一键值最多只需要2到4次磁盘IO
    高度平衡性
        不管如何变化，插入新的键值后，B+树依然会保持平衡，但是为了保持平衡，B+树可能会做大量的拆分页操作，
            页的拆分意味着磁盘操作，为了减少页的拆分操作，B+树提供了类似平衡二叉树的旋转功能
        删除键值后依旧能保持平衡，因为B+树使用填充因子来控制删除后的变化
分类
    聚集索引 和 非聚集索引，不同的是 叶子节点是否存放一整行的信息
```
